# this、call、apply、bind

`this` 是 JavaScript 面试里的高频考点，本质上考的是一句话：

**普通函数的 `this` 由调用方式决定，箭头函数的 `this` 由定义位置决定。**

---

## 1. 对 `this` 的理解

`this` 不是在函数定义时写死的，大多数情况下它会在**函数真正被调用时**确定指向。

先记住一条总原则：

- 普通函数：看调用位置
- 箭头函数：看外层作用域

对于普通函数，常见的绑定规则有 4 种，优先级通常是：

`new 绑定` > `显式绑定` > `隐式绑定` > `默认绑定`

需要注意：**箭头函数不参与这 4 条规则**，它没有自己的 `this`。

### （1）默认绑定

函数独立调用时，通常会走默认绑定。

```js
function foo() {
  console.log(this);
}

foo();
```

说明：

- 浏览器普通脚本中，非严格模式下通常指向 `window`
- 严格模式下，普通函数的 `this` 是 `undefined`
- 不同运行环境的顶层 `this` 可能不一样，不能一概而论

```js
'use strict';

function foo() {
  console.log(this);
}

foo(); // undefined
```

### （2）隐式绑定

当函数作为对象的方法被调用时，`this` 指向**最后调用它的那个对象**。

```js
const obj = {
  name: 'Alice',
  sayHi() {
    console.log(this.name);
  },
};

obj.sayHi(); // Alice
```

链式调用时看“点号左边最近的对象”：

```js
const obj1 = {
  name: 'obj1',
  obj2: {
    name: 'obj2',
    fn() {
      console.log(this.name);
    },
  },
};

obj1.obj2.fn(); // obj2
```

### （3）隐式绑定丢失

这是面试里最容易追问的点。

```js
const obj = {
  name: 'Alice',
  sayHi() {
    console.log(this.name);
  },
};

const fn = obj.sayHi;
fn();
```

原因：

- `fn()` 已经不是“作为 `obj` 的方法调用”
- 调用位置变了，`this` 也就变了

类似场景还包括：

- 回调函数单独传递：`setTimeout(obj.sayHi, 1000)`
- 事件回调中直接解构方法后再调用
- 将对象方法赋值给变量后再执行

常见解决方案：

- 使用箭头函数包一层
- 使用 `bind` 显式绑定
- 确保以 `obj.fn()` 的形式调用

### （4）显式绑定

可以通过 `call`、`apply`、`bind` 显式指定 `this`。

```js
function greet(city, job) {
  console.log(`${this.name} 来自 ${city}，职业是 ${job}`);
}

const person = { name: 'Bob' };

greet.call(person, '上海', '前端');
greet.apply(person, ['北京', '工程师']);

const boundFn = greet.bind(person, '杭州');
boundFn('开发');
```

结论：

- `call`：立即调用，参数逐个传
- `apply`：立即调用，参数以数组或类数组形式传
- `bind`：不立即调用，返回一个新函数

### （5）`new` 绑定

使用 `new` 调用构造函数时，会创建一个新对象，并让函数内部的 `this` 指向这个新对象。

```js
function Person(name) {
  this.name = name;
}

const p = new Person('Alice');
console.log(p.name); // Alice
```

`new` 调用大致会做 4 件事：

1. 创建一个新对象
2. 新对象连接到构造函数的 `prototype`
3. 构造函数内部的 `this` 指向这个新对象
4. 如果构造函数没有返回引用类型值，则返回这个新对象

### （6）箭头函数的 `this`

箭头函数没有自己的 `this`，它会捕获**定义时外层作用域**的 `this`。

```js
const obj = {
  name: 'Charlie',
  normal() {
    console.log(this.name);
  },
  arrow: () => {
    console.log(this.name);
  },
};

obj.normal(); // Charlie
obj.arrow(); // 通常不是 Charlie
```

为什么？

- 对象字面量本身不会创建新的 `this`
- `arrow` 的 `this` 来自它定义时的外层，而不是调用它的 `obj`

所以：

- 箭头函数**不适合**直接写成对象字面量方法
- 箭头函数很适合写在回调里，避免丢失外层 `this`

```js
function Timer() {
  this.count = 0;

  setInterval(() => {
    this.count++;
    console.log(this.count);
  }, 1000);
}
```

### （7）几个高频易错点

#### 1. 箭头函数不能用 `call`、`apply`、`bind` 改变 `this`

```js
const obj = { name: 'Alice' };

const fn = () => {
  console.log(this);
};

fn.call(obj); // this 不会变成 obj
```

原因是箭头函数的 `this` 在定义时就确定了。

#### 2. `bind` 绑定后，继续 `bind` 基本不会改掉原来的 `this`

```js
function foo() {
  console.log(this.name);
}

const fn1 = foo.bind({ name: 'A' });
const fn2 = fn1.bind({ name: 'B' });

fn2(); // A
```

后一次 `bind` 还能继续预置参数，但**不能把前一次绑定好的 `this` 再改掉**。

#### 3. `new` 调用绑定函数时，会忽略 `bind` 传入的 `this`

```js
function Person(name) {
  this.name = name;
}

const BoundPerson = Person.bind({ name: 'outer' }, 'Alice');
const p = new BoundPerson();

console.log(p.name); // Alice
```

这里真正生效的是 `new` 绑定，而不是 `bind` 时传入的对象。

#### 4. 顶层 `this` 依赖运行环境

常见结论：

- 浏览器普通 script 顶层：通常是 `window`
- ES Module 顶层：`undefined`
- Node.js CommonJS 顶层：通常是 `module.exports`

所以面试回答时不要把“顶层 `this` 一定等于 `window`”说死。

## 2. `call()`、`apply()`、`bind()` 的区别

三者的共同点：

- 都是函数原型上的方法
- 都可以改变普通函数执行时的 `this`
- 都可以实现“借用别的对象的方法”

核心区别如下：

| 方法 | 是否立即执行 | 参数形式 | 返回值 | 典型场景 |
| --- | --- | --- | --- | --- |
| `call` | 是 | 逐个传参 | 函数执行结果 | 参数个数明确时 |
| `apply` | 是 | 数组或类数组 | 函数执行结果 | 已有参数列表时 |
| `bind` | 否 | 先传一部分参数，后续再补 | 新函数 | 回调绑定、柯里化 |

示例：

```js
function sum(a, b, c) {
  return a + b + c;
}

console.log(sum.call(null, 1, 2, 3)); // 6
console.log(sum.apply(null, [1, 2, 3])); // 6

const plusOne = sum.bind(null, 1);
console.log(plusOne(2, 3)); // 6
```

补充说明：

- `apply` 的第二个参数不只是数组，也可以是类数组对象
- 现代开发里很多 `apply` 场景可以用扩展运算符 `...` 替代

```js
const nums = [1, 2, 3];

sum.apply(null, nums);
sum.call(null, ...nums);
```

### （1）`call` 的使用场景

适合参数数量明确、想立即执行的场景。

```js
function introduce(age) {
  console.log(`我是 ${this.name}，今年 ${age} 岁`);
}

const person = { name: 'Alice' };
introduce.call(person, 18);
```

### （2）`apply` 的使用场景

适合手里已经拿到数组、`arguments`、`NodeList` 这类参数集合时使用。

```js
function getMax() {
  return Math.max.apply(null, arguments);
}

console.log(getMax(1, 3, 2)); // 3
```

### （3）`bind` 的使用场景

适合不想立刻执行、而是想生成一个 `this` 固定的新函数。

```js
const button = {
  text: '提交',
  click() {
    console.log(this.text);
  },
};

const handler = button.click.bind(button);
setTimeout(handler, 1000);
```

## 3. 手写 `call`、`apply`、`bind`

下面的实现用于理解原理，适合面试回答。

需要先说明一个边界：

- 这些代码是**简化版实现**
- 重点在理解“临时挂载函数并执行”的思路
- 它们**不完全等价**于原生实现，尤其是严格模式函数、内建函数、`length`/`name` 属性等细节

### （1）手写 `call`

实现思路：

1. 判断调用者是不是函数
2. 处理 `thisArg`
3. 把当前函数临时挂到目标对象上
4. 执行后删除临时属性

```js
Function.prototype.myCall = function (context, ...args) {
  if (typeof this !== 'function') {
    throw new TypeError('myCall must be called on a function');
  }

  const thisArg = context == null ? globalThis : Object(context);
  const fnKey = Symbol('fn');

  thisArg[fnKey] = this;
  const result = thisArg[fnKey](...args);
  delete thisArg[fnKey];

  return result;
};
```

说明：

- `context == null` 同时兼容 `null` 和 `undefined`
- `Object(context)` 可以把字符串、数字、布尔值包装成对象
- 用 `Symbol` 可以避免覆盖原对象已有属性

### （2）手写 `apply`

和 `call` 的核心差别只有一个：参数接收方式不同。

```js
Function.prototype.myApply = function (context, args) {
  if (typeof this !== 'function') {
    throw new TypeError('myApply must be called on a function');
  }

  const thisArg = context == null ? globalThis : Object(context);
  const fnKey = Symbol('fn');
  const argList = args == null ? [] : Array.from(args);

  thisArg[fnKey] = this;
  const result = thisArg[fnKey](...argList);
  delete thisArg[fnKey];

  return result;
};
```

这里用 `Array.from(args)` 的原因是：

- 可以同时处理数组
- 也可以处理类数组对象

### （3）手写 `bind`

`bind` 最关键的点有两个：

- 返回的是一个新函数
- 新函数既要支持预置参数，也要兼容 `new` 调用

```js
Function.prototype.myBind = function (context, ...bindArgs) {
  if (typeof this !== 'function') {
    throw new TypeError('myBind must be called on a function');
  }

  const targetFn = this;

  function boundFn(...callArgs) {
    const isNew = this instanceof boundFn;
    const thisArg = isNew ? this : context;

    return targetFn.apply(thisArg, bindArgs.concat(callArgs));
  }

  if (targetFn.prototype) {
    boundFn.prototype = Object.create(targetFn.prototype);
    Object.defineProperty(boundFn.prototype, 'constructor', {
      value: boundFn,
      writable: true,
      configurable: true,
    });
  }

  return boundFn;
};
```

### （4）验证手写版本

```js
function greet(greeting, punctuation) {
  console.log(`${greeting}, ${this.name}${punctuation}`);
}

const person = { name: 'Alice' };

greet.myCall(person, 'Hello', '!');
greet.myApply(person, ['Hi', '!!']);

const boundGreet = greet.myBind(person, 'Hey');
boundGreet('?');
```

### （5）`bind` 遇到 `new` 的验证

```js
function Person(name, age) {
  this.name = name;
  this.age = age;
}

const BoundPerson = Person.myBind({ name: 'ignore' }, 'Alice');
const p = new BoundPerson(18);

console.log(p.name); // Alice
console.log(p.age); // 18
console.log(p instanceof Person); // true
```

## 4. 面试速记

如果面试官只想听结论，可以直接这样答：

1. `this` 的指向主要取决于函数的调用方式，不取决于书写位置。
2. 普通函数有 4 种常见绑定规则：默认绑定、隐式绑定、显式绑定、`new` 绑定，优先级通常是 `new` > 显式 > 隐式 > 默认。
3. 箭头函数没有自己的 `this`，它会继承外层作用域的 `this`，所以 `call`、`apply`、`bind` 无法修改箭头函数的 `this`。
4. `call` 和 `apply` 都会立即执行，区别在于参数形式不同；`bind` 不会立即执行，而是返回一个绑定后的新函数。
5. `bind` 返回的函数如果被 `new` 调用，会忽略绑定时传入的 `this`，但预置参数仍然有效。
