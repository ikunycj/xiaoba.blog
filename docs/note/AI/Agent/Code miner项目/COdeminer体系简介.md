这篇文档《agent-architecture-qa.md》是关于AI智能体（Agent）架构和系统设计的综合性问答手册。它结合微信支付内部的“Code-Miner”项目（一个代码分析智能体）进行详细阐述，内容深入且结构化，旨在帮助读者（特别是面试者）理解复杂Agent系统的核心概念、工程实现和设计决策。

以下是文档的主要内容概述：

### 1. 核心主题与架构

文档围绕Agent系统的几个核心层面展开：

- **Agent架构**：深入对比了**Agent**与**Workflow**（或Pipeline/DAG）的区别，并解释了Code-Miner为何选择基于**ReAct Loop**的Agent架构来处理不确定性高、需要动态决策的大规模代码分析任务。
    
- **企业级设计**：详细阐述了如何设计可扩展、安全、可观测的企业级Agent系统，包括分层架构（BFF/Agent Core/Tool Server/Worker）、安全隔离、权限审计、日志监控等。
    
- **MCP协议**：重点介绍了**Model Context Protocol (MCP)**，这是一个由Anthropic推出的标准化协议，用于连接AI模型与外部工具/数据源。文档解释了MCP在解决工具生态“N×M”集成问题上的价值，并详述了其分类、开发方法、传输层（stdio/SSE/WebSocket）以及在Code-Miner中的具体应用。
    
- **记忆与上下文**：探讨了在多轮对话和长任务中如何管理**短期记忆**、**长期记忆**和**任务记忆**，以及如何处理超出模型上下文窗口（Token限制）的问题，介绍了滚动总结、状态提取、RAG检索等策略。
    
- **模型原理**：解释了与Agent推理效率密切相关的底层模型原理，如**KV Cache**、**Prefix Caching**如何优化Attention计算，**Few-shot**提示在评测中的作用，以及**Attention机制**和**掩码（Mask）**​ 的工作原理。
    

### 2. 对现代Agent产品的分析

文档分析了如Claude Code、Codex、Manus等现代Agent产品，指出它们与传统聊天机器人的本质区别在于从“提供建议”转向“自主执行”，并具备丰富的工具调用、项目上下文感知和实际修改/执行能力。

### 3. 训练与对齐

虽然Code-Miner项目主要关注推理，但文档也涵盖了LLM训练的基础知识，包括**预训练**、**有监督微调**、**LoRA/QLoRA**高效微调技术，以及**RLHF**对齐流程，为读者提供了完整的模型生命周期视角。

### 4. 文档特点

- **实战导向**：每个知识点都紧密结合Code-Miner项目的实际代码和架构选择进行说明。
    
- **面试导向**：提供了大量“面试回答模板”和“深度追问应对”，直接面向技术面试场景。
    
- **系统全面**：从高层架构设计到底层模型原理，从外部协议集成到内部记忆管理，覆盖了构建生产级Agent系统所需考虑的主要方面。
    

**总结**：这份文档是一部关于AI Agent系统工程的“迷你百科全书”，它不仅解释了“是什么”和“为什么”，还通过具体的项目实践展示了“怎么做”，非常适合希望深入理解或构建复杂Agent系统的开发者、架构师和研究人员。
